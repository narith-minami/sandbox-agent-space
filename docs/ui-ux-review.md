# UI/UX レビュー: Coding Sandbox

日付: 2025年2月7日
対象画面: Create Sandbox

---

## 1. 左サイドバーの情報設計

### 問題点
- **冗長なリポジトリ名**: "sandbox-agent-space"が垂直方向に繰り返され、視覚的ノイズになっている
- **セッション種別の不明確さ**: 一覧に表示されているものがPRなのかタスクなのか判別困難
- **ステータス表示の一貫性欠如**: Merged/Closedの色とアイコンの関連性が弱く、一目で状況が把握しにくい

### 改善案
- **リポジトリグループ化**: リポジトリ単位で折りたたみ可能なセクションに整理
- **タイプアイコン導入**: PR、タスク、デバッグなどセッション種別をアイコンで視覚化
- **相対時間表示**: "Today 14:47" → "2時間前"のように、より直感的な表記に変更

---

## 2. フォームのエラーハンドリング

### 問題点
- **エラーの距離感**: グレーアウトしたStartボタンとエラーメッセージが物理的に離れており、関連付けが困難
- **解決方法の曖昧さ**: "Check OpenCode provider settings"という指示だが、Settingsへの導線が明示されていない
- **状態の不明確さ**: モデル未選択と選択肢がない状態の区別がつかない

### 改善案
- **インライン導線**: エラーメッセージ内にSettingsへの直接リンクを設置
- **インタラクティブフィードバック**: Startボタンホバー時に「モデルを選択してください」ツールチップを表示
- **空状態の明示**: ドロップダウンに「利用可能なモデルがありません」と表示

---

## 3. 入力エリアのUX

### 問題点
- **抽象的なプレースホルダー**: "Describe what you want done..."では具体的に何を書けば良いか不明確
- **情報の分離**: "Saved as Markdown"の注釈が入力欄から離れており、関連性が弱い
- **アイコンの曖昧さ**: マイクアイコンの機能（音声入力？）が不明確

### 改善案
- **具体例の提示**: プレースホルダーに「例: Implement user authentication using NextAuth.js」と記載
- **注釈の統合**: Markdown注釈を入力欄の下部枠線内に配置
- **ツールチップ追加**: マイクアイコンに「音声でタスクを入力」などの説明を追加

---

## 4. スペース活用

### 問題点
- **未使用空間**: 右下「No active session」エリアが画面の40%以上を無駄にしている
- **機会損失**: 新規ユーザーに対するオンボーディング要素が不足

### 改善案
- **クイックアクセス**: 最近のセッションやテンプレートをサムネイル表示
- **入力支援の追加**: タスク詳細入力を支援する要件チェックリストやプロンプト例を表示
- **ガイド表示**: 初回利用時にステップバイステップのチュートリアルを表示

---

## 5. ナビゲーションの明確化

### 問題点
- **重複する機能**: ヘッダー「History」と左上「Sessions」の役割分担が不明確
- **不明瞭なUI要素**: Sessions横の矢印（←）の機能が直感的でない

### 改善案
- **ラベル統一**: Sessionsを「Recent Sessions」にリネームして差別化
- **不要要素の削除**: 矢印アイコンを削除し、シンプルなラベルのみにする
- **現在位置の明示**: どの画面にいるかをハイライトやパンくずリストで示す

---

## 6. フォームレイアウトの階層性

### 問題点
- **フラットな構成**: リポジトリ→ブランチ→モデル→名前の4つのドロップダウンが横並びで、入力順序の誘導が弱い
- **視覚的グルーピング不足**: どの項目が関連しているかの視覚的ヒントがない

### 改善案
- **ステップ形式**: 段階的な入力フロー（1. リポジトリ選択 → 2. ブランチ選択 → ...）に変更
- **視覚的接続**: 矢印や番号付きラベルで入力順序を明示
- **カテゴリ分け**: ソース関連（リポジトリ・ブランチ）と設定関連（モデル・名前）をグループ化

## 実装時の注意点

1. **日本語UIとの整合性**: プレースホルダーやツールチップは日本語で統一
2. **アクセシビリティ**: ツールチップはキーボード操作でも表示可能に
3. **レスポンシブ対応**: モバイル表示時のフォームレイアウトを考慮
